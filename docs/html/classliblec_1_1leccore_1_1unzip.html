bool liblec::leccore::unzip::result ( unzip_log &  log,
std::string &  error 
)

The result of the unzipping operation.

Parameters
logUnzip log as defined in the unzip_log type. This may contain error messages for individual entries regardless of whether the method returns true or false. For example if one of the files already existed and was marked as read-only the unzipping operation continues and this method will return true yet an error will have actually occurred for that one entry.
errorError information.
Returns
Returns true if the operation was successful, else false. When false the error information is written back to error.

Start zipping operation.

Parameters
filenameThe full path to the zip archive, including the extension.
directoryThe directory to extract the zip archive to. Use an empty string to extract to the current directory.

This method returns almost immediately. The actual unzipping is executed on a seperate thread. To check the status of the unzipping call the unzipping method.

bool liblec::leccore::unzip::unzipping ( )

Check whether the unzipping operation is still underway.

Returns
Returns true if the unzipping is still underway, else false.

After calling start call this method in a loop or a timer, depending on your kind of app, then call result once it returns false.
